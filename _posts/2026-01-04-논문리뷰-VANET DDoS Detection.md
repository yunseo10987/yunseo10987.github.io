## 🦥 본문

## 초록

VANETs : 차량 사물 통신. 차량의 위치가 빈번하게 변하는 동적 네트워크. 

DDoS 공격을 탐지하기 위한 새로운 매커니즘인 신뢰 기반 프레임워크 제안. 

- 신뢰 평가 요소 : 주행 빈도 통계, 신뢰 가설 통계, 잔여 에너지, 신뢰 정책 및 데이터 요인 → 신뢰 평가 행렬
- 신뢰 평가 매커니즘 + 클러스터링 기법 = 저지 설계

## 도입

![image.png](attachment:26fd291e-fbb4-42a7-9ce8-87b90cb64f59:image.png)

VANET : 무선 네트워크를 사용하여 RSU(도로변 기지국) 또는 도로 인프라를 통해 인접 차량 간의 통신이 이루어지는 인프라 구조

- 짧은 시간동안만 연결과 해제 → 동적
- 이미 건설된 고속도로 및 도로는 차량의 패턴을 예측
- 하지만 운전자 행동, 교통 밀도 등은 불확실성
    - 밀도는 시간대 , 인구, 지역에 따라 다름
- VANET 노드들은 서로 다른 RSU로부터 데이터를 받고 멀티 홉이 발생할 수 있음
    
    → DoS, DDoS 취약. 왜 취약하냐면 10개의 노드를 거친다고 했을 때 1개의 노드만 점령해도 과부화가 됨. 또는 패킷 하나가 10개에 과부화를 줄 수도 있음 
    
    - DoS는 중복된 정보와 메시지로 패킷 범람을 통해 수행
    - DDoS는 좀비 PC나 자동화된 공격자에 의해 수행

차량 내 : AU(애플리케이션 장치) + OBU(차량 탑재 장치)

- 기술 평가 지표
    - AL(평균 지연 시간), PDR(패킷 전달률), DR(탐지율), EC(에너지 소비량) 등..
        - PDR : 목적지에 성공적으로 전송된 패킷의 수 판단

## 관련 연구

- DSRC 및 철회 방법 : 공격자 전송 기반.
    - 대상 노드가 메시지를 수신할 때, 메시지 전송을 위해 다양한 슬롯 시간과 다양한 위치에서 메시지가 획득되는 방식. 공격자는 슬롯 시간을 변경하고 다양한 차량 노드와 통신하려고 시도.
        - DSRC는 시간 슬롯이 있지만, 공격자는 짧은 시간 안에 많은 양의 데이터를 보내기 때문에 슬롯 시간을 조작함.
        - 슬롯 : 데이터 충돌을 막기 위해 시간을 아주 잘게 쪼개서 **"0.01초 동안은 A 차량만 말해, 그다음 0.01초는 B 차량만 말해"**라고 약속을 정해두는데, 이 쪼개진 시간 한 칸
        - 해킹을 통해 피해자의 차량 내에서 슬롯 대기 시간을 0으로 조작하여 계속해서 데이터를 보냄
    - DSRC를 4개의 클래스를 생성하고 우선순위 기반 정렬. 클래스 1이 가장 높음
        - 네트워크가 마비되면 중요한 메시지(브레이크 등)를 받아야 하기 때문에 클래스를 나눠 우선순위 정렬을 함.
    - 노드들이 특정 타임스탬프에서 보안 메시지를 수락하여 노드가 공격받았음을 인식하여 공격으로부터 보호
        - 특정 노드들이 특정 타임스탬프에서 보안 메시지를 받지 못한다면 공격자가 가짜 데이터를 쏟아부어 방해한다는 뜻이므로 공격받고 있음을 인지
- 트래픽 용량 및 블롬 필터를 사용 : 네트워크 주소 스푸핑을 방지
    - 트래픽 노출 측정
        1. 데이터 수집 및 분리
        2. 해당 데이터를 넣고 악성 노드의 존재 여부를 확인 DB에 저장
            - 악성 노드가 식별되면 알람을 발생. VANET 내 모든 노드에 해당 정보를 전송
        3. 해시 함수를 통해 블룸 필터 절차 수반. 
- APDA(공격 패킷 탐지 알고리즘) 및 MVND(악성 노드 탐지 알고리즘)
    - APDA : 악성 노드 인식을 위해 위치, 속도, 타임스탬프 취득. 확증 시간 이전에 위험 노드를 식별하여 오버헤드 중단을 최소화
    - MVND 방법론 : 하이브리드 네트워크를 적용하여 검증 시간 전에 악성 노드 확정
- 신뢰 관리 방식
    - 신뢰 평가 모델
        - 정책 제어 : 진입 신뢰와 데이터 신뢰
        - 능동적 신뢰 : 노드 차량의 과거 통신 이력
        - 소셜 신뢰 : 인접한 이웃 차량 평가를 고려하여 다른 차량의 신뢰 설정
    - 와치독 알고리즘을 사용한 탐지 매커니즘
        - 소스 차량으로부터 패킷을 전송받은 인접 이웃 노드들이 모니터링되며, 신뢰 값이 측정
        - 높은 신뢰도의 차량과 통신. 통신 문제가 발생하면 신뢰 감소 및 신뢰 테이블 업데이트
        - 단점 : 네트워크에서 충돌을 일으킴. 모니터링 이력이 방대하여 대규모 네트워크로 확장하기에는 어려움
    - 통신 사고 보고를 통해 신뢰성 측정
        - 크라우드 소싱 기능을 통해 신뢰 평가
            - 각각 차량의 신뢰값 브로드 캐스트는 전역 뷰에 의해 일어남
            - 고유 ID와 공개 키 기반 구조(PKI)를 포함하여 보안 강화
    - 노드 간의 협력적 방법을 사용하여 신뢰 토큰 생성.
        - 대칭 및 비대칭 접근 방식의 암호화 매커니즘 사용
        - 이웃 와치독 알고리즘 사용

## 문제 정의

Flooding 공격을 통한 자원 고갈 DDoS  공격. 분산적이므로 추적이 힘듦  

### V2V

서로 다른 위치에 있는 공격자들이 타임 슬롯을 사용하여 flooding. 

### V2I

RSU에 flooding. 자동화된 공격이 증가하여 네트워크에서 Denial 공격을 수행. 도구들을 쉽게 구할 수 있음.  

## 신뢰 기반 프레임워크

신뢰 평가 매커니즘 + 클러스터링 방식으로 공격자의 분류와 예측을 쉽게 함. 클러스터링은 신뢰 점수를 사용하여 수행  

각 노드의 신뢰 점수를 결정하고 Dominant Cluster(우세 클러스터)를 찾기 위해 신뢰 정책을 정의.

패킷 재전송을 방지하여 PDR을 높이고 EC를 최적화하기 위해 고도로보안된 Dominant Cluster를 식별. 

동적 클러스터링 구조를 구현하기 위해 신뢰 알고리즘이 제안. 신뢰 알고리즘들은 신뢰할 수 있는 경로를 식별하는 역할. Genetic Algorithm(유전 알고리즘)의 결합.

- 유전 알고리즘 : 수직 및 수평으로 성장하는 계층적 트리 구조
    - 수직적 성장 : 획득한 신뢰 점수로부터 얻은 TR(임계값) 보다 Heterogeneity(이질성)이 높을 때, 리프 노드를 두 개 추가
    - 수평적성장 : 최하위 레벨 노드들의 최적 서브 클러스터 개수가 정력적으로 계산. 모든 리프 노드의 이질성이 TR보다 작아질 때까지 계속
        - 이질성 : 주행 빈도와 엔트로피로부터 얻은 점수의 편차를 통해 계산

각 계층 수준에서 신뢰할 수 있는 노드들은 우세 클러스터로 인식되며 라우팅에 참여 가능   

클러스터링 단계는 DGSOT(동적으로 성장하는 자기 조직화 트리)를 통해 계층적 트리 구조를 찾는 데 활용 

### 신뢰 평가 매커니즘

신뢰는 계층적 트리 내에서 노드의 위치 결정. 악성 활동을 인식하고 마주칠 확률과 위험 요소를 구별 

- 신뢰 : 피어 노드가 해당 노드와의 여러 상호작용에서 신뢰할 수 있는 지 파악된 정도
- 기본은 직접 관찰.
- 직접적인 상호작용이 없었던 노드 사이의 신뢰를 확정하기 위해서는 중간 노드들의 간접 정보가 활용.
- 신뢰 평가 요인
    - 신뢰 가설 통계, 주행 빈도 통계, 데이터 요인, 유도된 신뢰 정책 및 클러스터 분리
    - 중심 클러스터를 파악하는 데 활용

### 신뢰 가설 통계

과거에 얼마나 성실하게 통신에 참여했는 지 수치화. 통신 성공은 신뢰 지수를 동적으로 증가 시키고 실패는 신뢰 지수를 감소.   

- 전송 추적 알고리즘 : 신뢰 구성 요소를 수치화하는 데 사용
- 가설 통계 알고리즘 : 경험 통계값을 수치화

### 주행 빈도 통계

주행 빈도 통계 : 특정 시점에서 데이터가 성공적으로 전송된 최고 빈도. 높은 빈도 통계값을 보유한채 단위 시간 당 높은 빈도의 데이터를 브로드캐스트. 단위 시간당 전체 성공적인 통신 빈도를 담당. 빈도 통계값은 알고리즘을 통해 수치화  

### 데이터 요인

데이터 요인 : 통신 과정 중 노드 i와 j 사이에서 데이터 패킷이 브로드 캐스트 되거나 분실되거나 수신될 가능성을 의미 

클러스터링 : 신뢰 기반 클러스터링 네트워크 모델에 초점  

### 클러스터 분리

노드들이 클러스터 됐을 때 모니터링이 효율적. 각 계층 수준에서 높은ㅇ 신뢰를 받는 노드는 네트워크 내 안전한 통신을 위한 우세 클러스터로 식별

초기 클러스터링은 신뢰 분석을 수행한 후 클러스터를 정제. 각 계층수준의 모든 노드는 Voronoi set 형성. 각 계층의 클러스터 분리(CS) 값은 두 중심점 사이의 최솟값에 대한 최댓값의 비율. CS를 화로용하여 모든 계층 수준에서 최적의 클러스터 수 달성    

- 신뢰 정책 : 모든 계층의 우세 클러스터는 역동적. 최대 신뢰 지수를 보유한 클러스터가 우세 클러스터
- 우세 클러스터 : 안전하고 가장 높은 THroughput을 보장.
- 신뢰 평가 행렬 : 데이터 베이스의 역할. 각 노드들의 신뢰 평가에 사용되는 정보 저장.
- 확장성은 노드들의 클러스터링을 통해 달성.
